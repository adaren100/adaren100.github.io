### Andrej Karpathy

Andrej Karpathy (formerly Tesla AI director, OpenAI founding team, Stanford PhD) announced he's joined Anthropic. He framed the move as a bet that the next few years at the LLM frontier will be "especially formative," and said he's excited to get back to R&D — while staying committed to resuming his education work over time ([link](https://x.com/karpathy/status/2056753169888334312)).

Andrej Karpathy (前 Tesla AI 总监、OpenAI 创始团队成员、Stanford 博士) 宣布加入 Anthropic。他给这一步的定性是：押注未来几年 LLM 前沿会"特别地塑形"，自己很期待回到 R&D 一线，同时也强调没有放弃教育这件事，会在合适的时间继续做下去 ([链接](https://x.com/karpathy/status/2056753169888334312))。

### Latent Space's Swyx

Latent Space's Swyx laid out his current 4-part AI SDLC. (1) Stand up roughly 50 tests with explicit instructions for the agent to add more, including a standing rule that browser e2e tests use computer vision to spot-check design and UX across mobile, desktop, iPad, and ultrawide resolutions. (2) Run a `/plan` pass to break up hot paths into isolated files and add proper logging plus error boundaries, treating the refactor as a maintainability and AI-editability cleanup. (3) Hand the plan back as a long-running execution with permission to break backward compatibility, with periodic commit / deploy / test checkpoints but instructions not to stop until all the work is done. (4) Spot-check the deployed app yourself and use `/steer` to redirect the agent when it drifts ([link](https://x.com/swyx/status/2056877529991205072)).

Latent Space 的 Swyx 把他现在跑 AI SDLC 的四步流程完整写了一遍。第一步：先搭好大约 50 条测试，并明确告诉 agent 之后还要继续往上加，其中有一条常驻规则——浏览器端 e2e 测试必须用 computer vision，在手机、桌面、iPad、超宽屏几个分辨率下都做一次视觉层面的设计和 UX 检查。第二步：跑一次 `/plan`，把热点路径拆成独立文件，顺手把 logging 和 error boundary 补齐，把这次重构直接当成一次"为可维护性 + 适合 AI 编辑"做的整理。第三步：把生成出来的 plan 重新塞回去让 agent 长跑，允许它打破向后兼容，期间按节奏自动 commit / deploy / test，但任务没做完不要停。第四步：你自己定期人肉抽查部署后的功能，发现跑偏就用 `/steer` 把它拉回来 ([链接](https://x.com/swyx/status/2056877529991205072))。
