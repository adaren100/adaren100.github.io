### Latent Space's Swyx

Latent Space's Swyx shared two takes worth flagging. First, his team ran a bake-off of Exa against competitors for agent web search, and within 1.5 hours unanimously converged on Exa — he points back to a pre-PMF interview with William Bryk as a case study in spotting companies on a generational tear ([link](https://x.com/swyx/status/2057180080078791036)). Second, he argues Sam Altman's mythical "build a business that gets better when models get better" is essentially what Swyx has been calling Agent Labs, and he's now seeing a direct correlation between model performance and agent-lab revenue, with a discontinuity in Q4 2025 ([link](https://x.com/swyx/status/2057119153337545096)).

Latent Space 的 Swyx 抛了两个值得记一下的观察。第一个是关于 agent 搜索：他们团队做了一次 Exa vs 竞品的 bake-off，1.5 小时内整个团队一致选 Exa；他顺势回去翻了一段当初 William Bryk 还没到 PMF 时的访谈，说这就是怎么提前识别"代际级"公司的范例 ([链接](https://x.com/swyx/status/2057180080078791036))。第二个观察是，他认为 Sam Altman 经典那句"做一门会随着模型变强而变强的生意"，本质上就是他之前一直在讲的 Agent Labs；现在 agent lab 的收入和模型性能之间已经出现非常直接的相关性，2025 Q4 还能看到明显的不连续跳变 ([链接](https://x.com/swyx/status/2057119153337545096))。

### Box CEO Aaron Levie

Box CEO Aaron Levie reports that token costs are about to become the dominant AI topic inside enterprises. After a dinner with a roomful of Fortune 500 CIOs, he says no one feels they've cracked it — companies are mixing strategies: routing workloads to different-tier models, granting better or worse agents based on user role, setting per-team spend caps, asking teams to justify AI by use case, or just giving unfettered access. Everyone is hunting for a semi-predictable cost model in a world where the underlying tech and pricing keep moving ([link](https://x.com/levie/status/2056965292753146019)).

Box CEO Aaron Levie 说 token 成本马上会成为企业内部关于 AI 最被反复讨论的话题。他刚跟一桌 Fortune 500 CIO 吃完饭，结论是：没人觉得自己找到了正确答案。大家在混搭各种策略——按 workload 路由到不同档位的模型、按用户角色发放更好或更差的 agent、按团队设置预算上限、要求各团队从用例角度证明 AI 投入合理、或者干脆全员放开用。所有人都在底层技术和定价不断变化的环境里，找一个"半可预测"的成本模型 ([链接](https://x.com/levie/status/2056965292753146019))。

### Every CEO Dan Shipper

Every CEO Dan Shipper unpacked the news that Anthropic acquired developer-tools startup Stainless (whose biggest customers were OpenAI and Google) by re-surfacing an October interview with Stainless CEO Alex Rattray on AI & I. The conversation reads as the design playbook Anthropic paid a reported $300M for. Highlights: (1) MCP servers should be lean and precise — keep the toolset small, give each tool a precise name and description, and apply a JSON filter on top to strip unnecessary data; (2) for larger APIs, switch to "dynamic mode" where the model only gets three meta-tools: list endpoints, pick one and learn it, then execute; (3) use MCP servers as a business copilot — Alex connects Notion and HubSpot so he can ask "which customers signed up last week?" and get one answer instead of multiple logins; (4) build a "company brain" by keeping Claude Code running and asking it to save customer feedback, SQL queries, and useful inputs into GitHub for the team to query later; (5) the future of MCP is code execution — instead of hundreds of tools, give the model a code-exec tool plus a doc-search tool, and let it write code against the SDK ([link](https://x.com/danshipper/status/2057122805657821240)).

Every 的 CEO Dan Shipper 借 Anthropic 收购开发者工具公司 Stainless（他们最大的客户是 OpenAI 和 Google）这条新闻，把自己十月份在 AI & I 上对 Stainless CEO Alex Rattray 的访谈又翻出来推了一遍——这段对话基本可以当作 Anthropic 花传闻中的 3 亿美金买来的设计 playbook 来读。几个亮点：(1) MCP server 要做得"瘦"且精确——工具数量保持小、每个工具要有精确的名字和描述，再在外面套一层 JSON filter 把多余字段过掉；(2) API 一旦变大，就切到 dynamic mode：模型只看到三个 meta 工具——列出 endpoints、挑一个并学习它、执行；(3) 把 MCP server 当业务 copilot 用——Alex 接上 Notion 和 HubSpot 之后，可以直接问"上周哪些客户注册了？"，省掉来回登录多个系统；(4) 用 Claude Code 给公司建一个"大脑"——让它常驻运行，把客户反馈、SQL 查询这类有用输入归档到 GitHub，团队之后随时检索；(5) MCP 的未来是代码执行——与其塞几百个工具，不如只给模型一个执行代码的工具加一个文档搜索工具，让它直接对着 SDK 写代码 ([链接](https://x.com/danshipper/status/2057122805657821240))。

### OpenAI

OpenAI announced a breakthrough on the planar unit-distance problem, an open question first posed by Paul Erdős in 1946. For nearly 80 years, mathematicians believed the best possible solutions looked roughly like square grids; an OpenAI model has now disproved that belief by discovering an entirely new family of constructions that performs better. OpenAI is positioning this as the first time AI has autonomously solved a prominent open problem central to a field of mathematics, and notably the proof came from a general-purpose reasoning model — not a system built specifically for math or for this problem ([link 1](https://x.com/OpenAI/status/2057176201782075690), [link 2](https://x.com/OpenAI/status/2057176203166171317)). Their broader framing: AI systems are becoming capable of holding long chains of reasoning, connecting distant fields, and surfacing paths researchers hadn't explored — abilities they expect to soon accelerate biology, physics, engineering, and medicine, while still depending on human judgment to choose the problems that matter ([link](https://x.com/OpenAI/status/2057176204541866087)).

OpenAI 宣布在 planar unit-distance 问题上取得突破——这是 Paul Erdős 在 1946 年提出的一个公开问题，将近 80 年里数学界都相信最优解大致长得像方形网格；一个 OpenAI 模型现在反证了这一假设，发现了一整族表现更好的全新构造。OpenAI 把这件事定位为"AI 首次自主解决一个数学领域中央层级的公开问题"，而且关键是证明出自一个通用推理模型，并非专门为数学或这一题训练的系统 ([链接 1](https://x.com/OpenAI/status/2057176201782075690)，[链接 2](https://x.com/OpenAI/status/2057176203166171317))。他们更大的叙事是：AI 已经能撑得起长链推理、把相距很远的领域串到一起、走出研究者没走过的路径——他们认为这些能力很快会加速生物、物理、工程、医学的进展，但仍然依赖人类判断来决定哪些问题值得做 ([链接](https://x.com/OpenAI/status/2057176204541866087))。
