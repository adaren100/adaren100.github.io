**[Codex-maxxing for long-running work](https://openai.com/index/codex-maxxing-long-running-work)** — _OpenAI · Jun 22_

**Main takeaway:** OpenAI published a whitepaper by Jason Liu on using Codex as a persistent workspace that preserves context and sustains progress across work that extends well beyond a single prompt.

**Main methods:**
- **Codex as persistent workspace.** Rather than treating Codex as a one-shot prompt tool, the guide frames it as a place where context and project state accumulate across sessions.
- **Breaking goals into verifiable steps.** The whitepaper emphasizes decomposing ambitious goals into steps whose completion can actually be checked, rather than vague open-ended instructions.
- **Maintaining continuity across workstreams.** Practical strategies are given for keeping multiple in-flight workstreams coherent over long time horizons.
- **Delegate vs. oversee.** A core theme is deciding when to hand execution fully to Codex versus when human review is most valuable.

**[Codex-maxxing：怎么用 Codex 撑起跑得很久的工作](https://openai.com/index/codex-maxxing-long-running-work)** — _OpenAI · 6月22日_

**Main takeaway:** OpenAI 发了一份 Jason Liu 写的白皮书，讲怎么把 Codex 当成一个能保留 context 的持续工作空间，撑住那些远超一次 prompt 就能搞定的长期项目。

**Main methods:**
- **Codex 当持续工作空间用。** 不是把 Codex 当一次性 prompt 工具，而是让它跨多个 session 累积 context 和项目状态。
- **把目标拆成能验证的步骤。** 白皮书强调把宏大目标拆成可以实际检查完成情况的小步骤，而不是给个模糊的开放式指令就完事。
- **跨多条 workstream 保持连续性。** 给出了一些实操方法，帮你在很长的时间跨度里让多条同时进行的工作线保持一致。
- **什么时候放手、什么时候盯着。** 核心主题之一是判断什么时候该把执行完全交给 Codex，什么时候人工 review 才是最有价值的。
