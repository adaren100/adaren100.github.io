**[OpenAI to acquire Ona](https://openai.com/index/openai-to-acquire-ona)** — _OpenAI · Jun 11_

**Main takeaway:** OpenAI plans to acquire Ona to expand Codex with secure, persistent cloud environments — the substrate Codex needs to run long-running enterprise coding agents.

**Main methods:**
- **What Ona brings.** Secure, persistent cloud environments — the kind of execution sandbox you need when an agent has to keep state across hours or days of work.
- **Codex roadmap.** The acquisition is positioned as the persistence/security layer for long-running AI agents inside enterprise workflows.
- **Strategic fit.** Pairs OpenAI's Codex model push with the infra piece (isolated, auditable runtime), where enterprise buyers most often get stuck.
- **Caveat.** Acquisition announcement; no terms, no closing date, no details about Ona's team or stack.

**[OpenAI 拟收购 Ona](https://openai.com/index/openai-to-acquire-ona)** — _OpenAI · 6月11日_

**Main takeaway:** OpenAI 准备收购 Ona，给 Codex 补上一层安全、持久的云环境——这是让 Codex 跑长时间企业级 coding agent 必需的底座。

**Main methods:**
- **Ona 带来了什么。** 安全且持久的云端执行环境——agent 跨小时甚至跨天保留状态的时候，就需要这种 sandbox。
- **Codex 的路线。** 这次收购被定位成长时间运行 AI agent 在企业流程里跑起来的"持久化 + 安全"那一层。
- **战略上的契合。** OpenAI 自己推 Codex 模型，缺的就是隔离可审计的运行时这块基础设施，而这正是企业客户最容易卡住的地方。
- **需要注意的 caveat。** 这是收购公告，没有交易条款也没有 closing date，没透露 Ona 团队和技术栈细节。
