**[Access OpenAI models and Codex through your Oracle cloud commitment](https://openai.com/index/openai-on-oracle-cloud)** — _OpenAI · Jun 10_

**Main takeaway:** Enterprises with Oracle Cloud commitments can now consume OpenAI models and Codex directly against that spend, with Oracle's enterprise security and governance layer in front.

**Main methods:**
- **Burn-down against existing commit.** Customers on Oracle Cloud can apply their existing committed spend to OpenAI usage — removes a procurement step for big enterprise buyers.
- **Codex included.** Not just chat models — Codex is part of the integration, aimed at enterprise coding workflows.
- **Oracle's enterprise wrapper.** Identity, governance, and compliance run through Oracle's stack, which is what regulated buyers tend to insist on.
- **Caveat.** Distribution / procurement announcement, not a new model or capability — interesting for buyers, not for researchers.

**[通过 Oracle 云的 commit 用 OpenAI 模型和 Codex](https://openai.com/index/openai-on-oracle-cloud)** — _OpenAI · 6月10日_

**Main takeaway:** 已经签了 Oracle Cloud commit 的企业，现在可以直接拿那部分预算来跑 OpenAI 模型和 Codex，前面套一层 Oracle 的企业安全和治理。

**Main methods:**
- **直接抵 commit。** Oracle Cloud 上的客户可以拿原本就要花掉的 commit 去消耗 OpenAI 用量——对大企业采购来说省了一步流程。
- **包含 Codex。** 不只是聊天模型，Codex 也在里头，瞄准的是企业 coding workflow。
- **套一层 Oracle 的企业壳。** 身份、治理、合规全走 Oracle 那一套，受监管行业客户通常会要求这种安排。
- **需要注意的 caveat。** 这是分发/采购通道的公告，不是新模型也不是新能力，对买家有意义，对研究人员意义不大。
